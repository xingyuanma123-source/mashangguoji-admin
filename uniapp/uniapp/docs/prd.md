# 司机报账微信小程序 - 需求文档

## 一、项目概述

物流公司内部使用的司机报账微信小程序，用于短驳司机提交每日费用报账。后续会做一个客服管理网页，共用同一个数据库。

**使用人群：** 公司短驳货运司机（约13人）
**核心场景：** 司机每天跑多趟短驳，每趟可能开不同的车。一天结束后统一提交当天所有车辆的费用报账。客服在网页端逐条确认后自动录入总表，确认前司机和客服都可以修改，确认后锁定不可修改。

## 二、核心业务流程

司机提交报账 → 状态：\"待确认\"
      ↓
司机可随时修改（客服也可以在网页端修改）
      ↓
客服在网页端逐条确认 → 状态变为\"已确认\"→ 自动进入总表
      ↓
已确认的记录锁定，不可再修改

**总表 = expense_records 表中 status=\"confirmed\" 的所有记录。** 客服可随时查看和导出，导出格式与现有 Excel 模板一致（每个司机一个 Sheet，按月汇总）。

## 五、各页面详细需求

### 页面1：登录页

**表单：** 账号 + 密码
**逻辑：** 查 drivers 表匹配 → 成功存缓存跳首页 → 失败提示\"账号或密码错误\"
**登录状态保持，除非手动退出。**

### 页面2：报账提交页（首页）⭐

**整体概念：** 一次提交 = 一天的报账。顶部选日期，下面是多个\"车辆卡片\"，每个卡片对应一辆车的一趟任务。最后一键提交。

---

#### 🔒 自动暂存功能

- **自动保存：** 每次修改任何字段后，自动将当前页面数据保存到本地缓存
- **恢复：** 打开报账页时，如有未提交的暂存数据，弹出\"检测到上次未提交的报账记录，是否恢复？\"→ 可选\"恢复\"或\"放弃\"
- **清除：** 提交成功后清除暂存数据

---

#### 页面布局

**顶部：**
- 司机姓名（自动显示，不可编辑）
- 日期选择器（默认今天）
- 是否加班：开关按钮（默认关闭）。打开 = 当天加班，向 overtime_records 表插入记录；关闭 = 取消加班，删除记录。切换日期时自动查询该天是否已标记加班并更新开关状态。

**中间：车辆卡片列表**

每个卡片包含：

**（a）车牌号 — 搜索式智能输入**
- 输入框，placeholder=\"输入车牌号搜索\"
- 输入时实时从 vehicles 表模糊匹配，下方显示匹配的车牌列表
- 例如司机输入\"桂FB6\"，下方弹出：
  ```

  桂FB6657
  桂FB6379
  桂FB6609
  桂FB6659
  桂FB6679
  ```
- 可点击选中，也可手动输入
- 校验：车牌在库中 → 绿色边框 ✅；不在库中 → 橙色边框 ⚠️ + \"该车牌不在公司车辆库中\"
- 不阻止提交，提交时统一弹确认框

**（b）路线/地点**
- 文本输入框，**非必填**
- placeholder=\"路线或地点（选填），如：越南—桂福\"

**（c）费用明细 — 下拉框 + 金额**

每条费用一行：
```
[下拉框：选择费用类型 ▼]    [金额输入框]   [×]
（选\"其他\"时多出：）
[请输入具体费用名称：____________]
```
- 下拉框选项从 fee_types 表读取
- 选\"其他\" → 必须填写具体名称
- 点\"+ 添加费用\"添加新行
- 每行可删除

**（d）凭证图片**
- 图片上传，非必填，最多9张

**（e）备注**
- 多行文本框，非必填

**（f）本车费用小计**
- 实时自动计算

**卡片操作：**
- 右上角可删除整个卡片
- 初始显示1个空卡片
- 底部\"+ 添加车辆\"按钮

---

**底部：**
- 今日费用合计：¥___ （实时汇总）
- \"提交报账\"按钮

**提交逻辑：**
1. 校验：每个卡片车牌不为空、至少一条费用、\"其他\"类型名称不为空
2. 车牌校验：有不在库中的车牌 → 弹确认框，确认后继续
3. 费用映射到对应字段，同类累加，\"其他\"写入 note_amount + note_detail
4. 每个卡片 → expense_records 插入一条，status=\"pending\"，commission=0
5. 成功 → \"提交成功，共X辆车\"，清空页面 + 清除暂存
6. 失败 → \"提交失败，请重试\"

## 六、全局规范

### 导航
底部 Tab 导航：报账、记录、我的。当前标签高亮。

### 配色
主色：蓝色或绿色。已确认=绿色，待确认=橙色，背景=浅灰，卡片=白色。

### 加载与空状态
请求时 loading，列表空显示\"暂无记录\"，网络错误提示\"网络异常，请重试\"。

### 数据权限
司机只能查看和修改自己的数据。

### 编辑权限
只有 status=\"pending\" 的记录可以编辑，status=\"confirmed\" 的记录锁定不可修改。

### 自动暂存
报账提交页填写内容实时保存到本地缓存，提交成功后清除。

## 八、重要提醒

1. **没有审核驳回机制**，只有\"待确认\"和\"已确认\"两种状态
2. **待确认的记录**司机和客服都可以修改，**已确认的记录**谁都不能改
3. **总表 = status=\"confirmed\" 的记录**，不是独立的表
4. **导出格式**与现有 Excel 模板一致：每个司机一个 Sheet，含12种费用列 + 备注 + 支出合计 + 提成 + 凭证图片 + 底部备用金汇总
5. **提成由客服填写**（确认前），司机端只读
6. **客服网页后续共用同一个数据库**，需要能：查看/修改/确认报账、管理车辆库、管理司机、充值备用金、查看和导出总表
7. **备用金余额**只统计已确认的支出：充值总和 − 已确认支出总和
