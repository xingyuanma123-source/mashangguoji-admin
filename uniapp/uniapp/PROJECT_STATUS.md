## 📝 重要说明

### 业务逻辑
1. **报账流程**：司机提交 → 待确认 → 客服确认 → 已确认（锁定）
2. **总表概念**：status="confirmed" 的所有记录即为总表
3. **编辑权限**：只有待确认的记录可以编辑，已确认的记录锁定
4. **加班统计**：按 record_date 去重后的天数，而非记录条数
5. **备用金余额**：只统计已确认的支出

### 技术特性
1. **自动暂存**：报账页面数据实时保存到本地缓存
2. **智能搜索**：车牌输入时实时匹配车辆库
3. **图片压缩**：自动压缩到1MB以内
4. **数据安全**：RLS 策略确保司机只能访问自己的数据
5. **实时计算**：费用合计自动计算
